# Security Guidelines

## 入力バリデーション

- すべてのユーザー入力をシステム境界でバリデーションする
- SQL インジェクション対策にはパラメータ化クエリを使用する
- XSS 対策として HTML を出力する際はエスケープ処理を行う
- シェルコマンドへのユーザー入力混入を避ける（コマンドインジェクション対策）

## 認証・認可

- パスワードは bcrypt などの強力なハッシュアルゴリズムで保存する
- JWT トークンの有効期限を短く設定し、リフレッシュトークンを使う
- 最小権限の原則（Principle of Least Privilege）を適用する

## 機密情報

- シークレット・APIキーは環境変数で管理し、コードに直接書かない
- `.env` ファイルを `.gitignore` に追加する
- ログに機密情報（パスワード、トークン、個人情報）を出力しない

## 依存関係

- 定期的に `npm audit` を実行して脆弱性を確認する
- 依存パッケージを最新の安定版に保つ
- lock ファイル (`package-lock.json`) をコミットして依存関係を固定する
